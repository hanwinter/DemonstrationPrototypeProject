一、修复路由返回问题

当前问题：
“就诊人管理”有两个入口：
1. 首页快捷入口进入“就诊人管理”
2. 我的页面进入“就诊人管理”

现在从“我的 → 就诊人管理 → 编辑就诊人”进入后，点击返回上一级时没有回到“我的”页面，而是直接回到了首页。这个属于路由跳转错误。

优化要求：
1. 返回逻辑不能写死跳转首页。
2. 从“我的”页面进入“就诊人管理”后：
   - 就诊人管理页面返回应回到“我的”页面；
   - 编辑就诊人页面返回应回到“就诊人管理”页面；
   - 再从就诊人管理返回时，应回到“我的”页面。
3. 从首页快捷入口进入“就诊人管理”后：
   - 就诊人管理页面返回应回到首页；
   - 编辑就诊人页面返回应回到“就诊人管理”页面；
   - 再从就诊人管理返回时，应回到首页。
4. 请优先使用现有路由机制处理来源页面，例如：
   - router.back()
   - 或通过 query 参数传递 from=home / from=mine
   - 或通过 route state / meta 记录入口来源
5. 不要影响其他页面已有跳转逻辑。
