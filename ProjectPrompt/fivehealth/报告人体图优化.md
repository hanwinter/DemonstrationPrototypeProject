只修改五健家长端报告页面中的“人体健康画像”区域，不修改页面顶部学生信息卡、综合评估卡、右侧异常说明卡片、底部导航和其他页面结构。

目标：
将当前页面中间卡片左侧的默认人体插画，替换为本地 PNG 图片，并根据学生性别动态显示男孩或女孩图片。

图片路径：
- 男孩图片：ProjectImage\男孩.png
- 女孩图片：ProjectImage\女孩.png

实现要求：
1. 读取当前学生信息中的性别字段，字段可能为 gender、sex 或 studentGender，优先使用当前页面已有的数据字段。
2. 当学生性别为“男”时，显示 ProjectImage\男孩.png。
3. 当学生性别为“女”时，显示 ProjectImage\女孩.png。
4. 当性别为空、未知或无法识别时，默认显示 ProjectImage\男孩.png。
5. 只替换人体底图，不要修改右侧“高风险 / 视力健康 / 左眼视力下降”等说明卡片内容。
6. 保留当前人体图上的异常点位展示能力，但异常点不要写死在图片中，应继续由前端以 absolute 定位方式叠加。
7. 如果当前异常点位是写在人体图组件内部的，需要继续保留，并让点位相对新图片容器定位。
8. 图片容器尺寸固定，建议宽度 160-180px，高度 260-300px，图片使用 object-fit: contain，避免图片变形或撑开卡片。
9. 图片背景应保持透明 PNG 的显示效果，不要增加额外边框。
10. 保持当前卡片整体布局：左侧人体图，右侧异常说明卡片，左右排列关系不变。

样式要求：
1. 图片在左侧区域水平居中、垂直居中。
2. 图片不要超过当前白色卡片高度。
3. 异常点仍然显示为彩色圆点，带白色描边和轻微阴影。
4. 异常点应覆盖在图片上层，z-index 高于图片。
5. 页面整体视觉风格保持当前蓝绿色、浅色医疗风格。

建议实现方式：
1. 新增一个 bodyImage 计算属性或函数，根据学生性别返回图片路径。
2. 在模板中将原人体 SVG / div 插画替换为 img 标签。
3. img 标签使用 :src="bodyImage"。
4. 外层保留 body-map-container 容器，用于承载图片和异常点。
5. 异常点使用 absolute 定位，相对于 body-map-container 定位。

参考代码结构：

const getBodyImage = (gender) => {
  if (gender === '女') return '/ProjectImage/女孩.png'
  return '/ProjectImage/男孩.png'
}

页面展示：
<div class="body-map-container">
  <img class="body-image" :src="bodyImage" alt="人体健康画像" />
  <div class="health-marker marker-vision"></div>
  <div class="health-marker marker-mouth"></div>
  <div class="health-marker marker-weight"></div>
  <div class="health-marker marker-spine"></div>
  <div class="health-marker marker-mental"></div>
</div>

注意：
- 不要删除异常点逻辑。
- 不要把异常点合并到 PNG 图片里。
- 不要修改其他页面模块。
- 不要重构整个页面，只做局部替换。
