继续优化五健家长端报告页面。

当前页面已经实现：
1. 点击右侧五健按钮，人体结构图对应区域高亮。
2. 点击右侧五健按钮，人体结构图下方摘要卡片切换显示当前健康项异常摘要。
3. 点击摘要卡片中的“查看详情 >”，页面会滚动定位到下方详情内容区。

现在需要调整第 3 点交互。

一、当前问题

“查看详情 >”点击后页面滚动到下方详情区，会破坏移动端阅读连续性。用户看完详情后，如果想继续查看其他五健项，需要再滑回上方点击其他按钮，造成反复上下滑动，体验不好。

二、修改目标

将“查看详情 >”从“页面滚动定位”改为“弹框展示详情”。

要求：
1. 点击人体健康画像卡片下方摘要中的“查看详情 >”，不再滚动页面。
2. 改为在当前页面打开详情弹框。
3. 弹框展示当前 activeHealthCode 对应健康项的完整详情。
4. 关闭弹框后，页面仍停留在当前五健健康画像区域，不改变页面滚动位置。
5. 保留当前右侧五健按钮切换逻辑。
6. 保留人体图 overlay 高亮逻辑。
7. 保留人体图下方摘要卡片切换逻辑。
8. 不修改顶部学生信息卡、综合评估卡、底部导航。

三、弹框形式

请使用移动端底部半屏弹层 Bottom Sheet，不要使用页面跳转。

弹框要求：
1. 从页面底部弹出。
2. 宽度 100%。
3. 最大高度建议 70vh - 80vh。
4. 内容超过高度时，弹框内部滚动。
5. 页面背景增加半透明遮罩。
6. 点击遮罩或右上角关闭按钮可关闭。
7. 弹框圆角建议：顶部左上、右上 20px - 24px。
8. 弹框打开后，禁止背景页面滚动。
9. 弹框关闭后，恢复页面滚动。
10. 底部导航可以被遮罩覆盖，不需要保持可点。

四、弹框内容结构

弹框展示当前选中健康项 activeHealth 的完整详情。

建议结构：

  </div>

五、不同状态展示规则

1. 高风险、中风险、低风险：
   - 正常展示异常指标、检查结果、参考范围、结果说明、健康建议。
2. 正常：
   - 异常指标可显示“未见异常”。
   - 健康建议显示日常保持建议。
3. 待填写：
   - 异常指标显示“心理量表未完成”。
   - 检查结果显示“暂无”。
   - 参考范围显示“完成量表后生成”。
   - 结果说明显示“心理量表尚未完成，暂不能形成心理健康综合判断。”
   - 健康建议显示“请完成心理健康量表，以便生成完整评估结果。”

七、交互状态

新增状态变量：

const showHealthDetailSheet = ref(false)

点击“查看详情 >”：
showHealthDetailSheet = true

关闭弹框：
showHealthDetailSheet = false

注意：
1. 不要再调用 scrollIntoView。
2. 不要再改变页面 scrollTop。
3. 不要跳转页面。
4. 不要展开下方详情区。
5. 只打开弹框展示当前 activeHealth 的详情。

八、样式要求

1. 弹框整体风格保持当前蓝绿色医疗风格。
2. 标题使用深色加粗。
3. 风险标签颜色根据状态变化：
   - 高风险：红色
   - 中风险：橙色
   - 低风险：蓝色
   - 正常：绿色
   - 待填写：紫色
4. 内容区域使用白底，分组清晰。
5. 每个 detail-section 之间留出 14px - 18px 间距。
6. 健康建议可以使用浅青色背景块突出。
7. 弹框内文字不要过小，正文建议 14px - 15px。
8. 关闭按钮点击区域不小于 36px。

九、保留内容

2. 但上方“查看详情 >”不再定位到该区域。
3. 下方详情区仍然作为报告完整内容保留，用户向下浏览时可以看到。
4. 后续如果确认弹框体验更好，再考虑删除或弱化下方详情区。
