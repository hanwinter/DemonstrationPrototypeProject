继续优化五健家长端报告页面。

上一版已经实现了左侧五健按钮点击、人体图对应区域 overlay 高亮、下方专项详情切换。但现在存在两个问题：
1. 人体图和按钮只体现“点击选中”，没有直接展示五健五项是否异常，业务价值不明显。
2. 点击按钮后，专项详情在页面下方，首屏内看不到，用户感知不到点击结果。

本次目标：
将“人体健康画像卡”升级为“五健状态总览 + 当前项摘要 + 人体区域高亮”的综合卡片。

修改范围：
只修改人体健康画像卡及其下方五健概览相关区域，不修改顶部导航、学生信息卡、综合评估卡、底部导航。

一、人体健康画像卡结构调整

将当前结构优化为：

1. 卡片标题：五健健康画像
2. 卡片主体：
   - 左侧：人体图，显示五健 overlay；
   - 右侧：五健状态按钮列表；
3. 卡片底部：
   - 当前选中健康项摘要；
   - 显示健康项名称、风险等级、异常摘要、查看详情入口。

推荐结构：

<div class="body-health-card">

二、五健状态按钮要求

右侧五健按钮不再只显示分类名，必须显示每个健康项的状态。

五项按钮内容：
1. 眼 / 视力 / 高风险
2. 体 / 体重 / 中风险
3. 脊 / 脊柱 / 低风险
4. 齿 / 口腔 / 正常
5. 心 / 心理 / 待填写

按钮样式：
1. 每个按钮高度建议 44px - 48px。
2. 当前选中按钮高亮。
3. 按钮状态颜色根据风险等级变化：
   - 高风险：红色
   - 中风险：橙色
   - 低风险：蓝色
   - 正常：绿色
   - 待填写：紫色
4. 未选中按钮背景要轻，不要太重。
5. 点击按钮后，当前按钮、人体图 overlay、底部摘要同步切换。

三、人体图 overlay 展示规则

当前不能只显示 activeHealthCode 的 overlay。
需要根据五健状态展示全部有意义的状态：

1. 高风险、中风险、低风险：对应 overlay 默认弱显示。
2. 当前选中的健康项：对应 overlay 强高亮。
3. 正常项：默认不显示 overlay，或仅极淡显示。
4. 待填写项：可以显示淡紫色虚线或极淡 overlay。
5. overlay 的意义是展示“五健状态分布”，不是只做点击反馈。

请实现 getOverlayClass(item) 逻辑：

- 如果 item.code === activeHealthCode：返回 active
- 如果 item.status 是 高风险 / 中风险 / 低风险：返回 visible
- 如果 item.status 是 待填写：返回 pending
- 如果 item.status 是 正常：返回 hidden

CSS 参考：

.health-overlay {
  position: absolute;
  z-index: 2;
  pointer-events: none;
  opacity: 0;
  transition: all 0.2s ease;
}

.health-overlay.visible {
  opacity: 0.35;
}

.health-overlay.active {
  opacity: 1;
  transform: scale(1.04);
}

.health-overlay.pending {
  opacity: 0.22;
  border-style: dashed;
}

.health-overlay.hidden {
  opacity: 0;
}

四、当前项摘要区

在人体图卡片底部新增“当前项摘要”，用于解决点击后首屏无反馈的问题。

摘要内容根据 activeHealthCode 动态切换。

示例：

activeHealthCode = vision：
标题：视力健康
风险：高风险
摘要：左眼视力下降
入口：查看详情 >

activeHealthCode = weight：
标题：体重健康
风险：中风险
摘要：BMI 偏高
入口：查看详情 >

activeHealthCode = mental：
标题：心理健康
风险：待填写
摘要：心理量表未完成，暂不能形成综合判断
入口：去填写 >

要求：
1. 摘要区必须放在人体图卡片内部。
2. 点击左侧/右侧五健按钮后，摘要区即时变化。
3. 摘要区文字不要太长，最多两行。
4. 如果有长说明，放到下方专项详情或点击“查看详情”后展示。
5. “查看详情”点击后，可以平滑滚动到下方专项详情卡片。
6. 如果暂时不实现滚动，按钮可以先保留样式。
