继续优化五健家长端报告页面中的“五健专项详情底部弹框”。

本次只修改点击“查看详情 >”后弹出的底部详情弹框内容结构和样式，不修改报告页其他区域，不修改五健按钮切换逻辑，不修改人体图 overlay 逻辑。

一、当前问题

当前底部弹框中存在以下问题：
1. 标题下方的“本次筛查结果详情”是冗余文案，建议删除。
2. “异常指标、检查结果、参考范围、结果说明、健康建议”每项都使用大卡片展示，信息过于松散，占用高度过大。
3. “检查结果”和“参考范围”是对照信息，适合左右两列展示。
4. “健康建议”当前放在最底部，但这是家长最关心的行动信息，建议上移。
5. “结果说明”更适合改名为“当前结论”，并放在靠前位置。

二、修改目标

将弹框结构调整为：

顶部：
健康项名称 + 风险等级标签

内容顺序：
1. 当前结论
2. 健康建议
3. 检查信息
   - 检查结果
   - 参考范围
   - 异常指标

不要继续使用多个大卡片堆叠展示。

三、弹框顶部调整

当前顶部类似：

口腔健康
本次筛查结果详情

调整为：

口腔健康                         正常

要求：
1. 删除“本次筛查结果详情”。
2. 健康项名称左侧展示，风险等级标签右侧展示。
3. 右上角关闭按钮保留。
4. 风险等级标签颜色根据状态变化：
   - 高风险：红色
   - 中风险：橙色
   - 低风险：蓝色
   - 正常：绿色
   - 待填写：紫色

推荐结构：

五、字段展示规则

1. 当前结论：
   - 使用 activeHealth.description。
   - 原“结果说明”字段不再单独展示，改为“当前结论”。

2. 健康建议：
   - 使用 activeHealth.advice。
   - 放在“当前结论”下面。
   - 使用浅青色背景突出，但不要高度过大。

3. 检查结果：
   - 使用 activeHealth.result。
   - 与参考范围左右两列展示。

4. 参考范围：
   - 使用 activeHealth.reference。
   - 与检查结果左右两列展示。

5. 异常指标：
   - 使用 activeHealth.abnormalValues。
   - 放在检查信息区底部，不要单独占一个大卡片。
   - 多个异常值用顿号连接。

六、正常场景展示示例

当 activeHealth 为口腔健康，状态为正常时，弹框展示：

标题：
口腔健康                      正常

当前结论：
本次口腔筛查未见明显异常。

健康建议：
继续保持良好刷牙习惯，定期口腔检查。

检查信息：
检查结果：未见异常
参考范围：正常
异常指标：未见异常

七、异常场景展示示例

当 activeHealth 为体重健康，状态为中风险时，弹框展示：

标题：
体重健康                      中风险

当前结论：
BMI 处于偏高区间，同时腰围和体脂率提示体重管理需要关注。

健康建议：
控制含糖饮料和高热量零食，增加中等强度运动。

检查信息：
检查结果：BMI：23.8；腰围：72cm；体脂率：偏高
参考范围：BMI：P85-P95；腰围：参考同年龄同性别标准
异常指标：BMI偏高、腰围偏高、体脂率偏高

八、样式要求

1. 弹框整体继续使用底部半屏 Bottom Sheet。
2. 弹框最大高度保持 70vh - 80vh。
3. 内容超出时在弹框内部滚动。
4. 不要再为每个字段使用一个大白色卡片。
5. 当前结论、健康建议、检查信息使用紧凑 section 结构。
6. 每个 section 之间间距控制在 14px - 18px。
7. 健康建议使用浅青色背景块突出。
8. 检查结果和参考范围使用左右两列布局。
9. 小屏幕下，如果左右两列内容过长，可以自动换行成上下展示。
10. 正文字号建议 14px - 15px。
11. 字段标题使用灰色小字或中等字重。
12. 重要内容使用深色。
13. 不要出现过多边框，整体保持清爽。
