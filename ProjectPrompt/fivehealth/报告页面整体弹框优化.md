继续优化五健家长端报告页面中的“五健专项详情底部弹框”。

本次只修改点击“查看详情 >”后弹出的底部详情弹框，不修改报告页主页面结构，不修改五健健康画像卡主体布局，不修改人体图 overlay 高亮逻辑，不修改底部导航。

一、当前弹框问题

当前弹框展示内容已经比较清楚，但仍有几个问题：

1. 弹框标题使用“体重健康”“口腔健康”等完整名称，标题略长，和主页面右侧五健按钮“体重、口腔、视力”等命名不统一。
2. 健康建议区域背景存在上下渐变或色带感，视觉不够稳定，建议改成纯浅色圆角提示卡。
3. 异常指标目前放在检查信息后方，阅读顺序不够合理。
4. 用户在弹框内只能查看当前专项，如果要查看其他专项，需要关闭弹框、重新点击其他五健按钮、再点查看详情，操作链路偏长。
5. 建议在弹框内增加五健专项切换功能，方便用户连续查看视力、体重、脊柱、口腔、心理五个专项详情。

二、修改目标

1. 弹框标题使用短名称：
   - 视力
   - 体重
   - 脊柱
   - 口腔
   - 心理

2. 健康建议区域改为纯浅色背景圆角卡片：
   - 去掉上下渐变效果；
   - 宽度与内容区对齐；
   - 增加小圆角；
   - 保持轻量、清爽。

3. 异常指标上移：
   - 放在“当前结论”下方；
   - 放在“健康建议”上方；
   - 用轻量信息条展示，不要单独做大卡片。

4. 弹框内新增五健专项切换标签：
   - 支持在弹框内切换视力、体重、脊柱、口腔、心理；
   - 切换后弹框内容立即更新；
   - 弹框不关闭；
   - 主页面 activeHealthCode 同步更新；
   - 背后页面五健按钮选中态和人体 overlay 同步变化。

四、弹框内增加五健专项切换标签

在标题区下方增加一行五健切换标签。

推荐展示：

[视力 高] [体重 中] [脊柱 低] [口腔 正常] [心理 待填]

或：

[视力] [体重] [脊柱] [口腔] [心理]

要求：
1. 使用 healthItems 循环渲染。
2. 点击标签后更新 activeHealthCode。
3. 当前选中的标签高亮。
4. 标签中可以显示健康项短名称和风险状态。
5. 标签支持横向滚动，避免小屏幕挤压。
6. 弹框不关闭。
7. 切换后“标题、风险等级、当前结论、异常指标、健康建议、检查信息”全部同步更新。
8. 背后主页面使用同一个 activeHealthCode，因此人体 overlay 和主页面按钮选中态也同步变化。

推荐结构：

<div class="sheet-health-tabs">
  <button
    v-for="item in healthItems"
    :key="item.code"
    class="sheet-health-tab"
    :class="{ active: item.code === activeHealthCode }"
    @click="activeHealthCode = item.code"
  >
    <span class="tab-name">{{ item.navName }}</span>
    <span class="tab-status">{{ getShortStatus(item.status) }}</span>
  </button>
</div>

getShortStatus 建议：
高风险 → 高
中风险 → 中
低风险 → 低
正常 → 正常
待填写 → 待填

九、数据字段要求

healthItems 中需要包含以下字段：

code
navName
name
status
summary
abnormalValues
result
reference
description
advice
overlayClass

其中：
1. navName 用于弹框标题和标签，例如“体重”。
2. name 可继续用于主页面或完整名称，例如“体重健康”。
3. abnormalValues 用于异常指标展示。
4. description 用于当前结论。
5. advice 用于健康建议。

十、弹框切换交互要求

1. 点击主页面摘要卡“查看详情 >”打开弹框。
2. 打开弹框时，默认展示当前 activeHealthCode 对应内容。
3. 在弹框内点击五健专项标签，更新 activeHealthCode。
4. activeHealthCode 更新后：
   - 弹框标题更新；
   - 风险等级更新；
   - 当前结论更新；
   - 异常指标更新；
   - 健康建议更新；
   - 检查信息更新；
   - 主页面右侧五健按钮选中态同步；
   - 人体图 overlay 同步。
5. 切换五健专项时，弹框不关闭。
6. 关闭按钮只关闭弹框，不改变 activeHealthCode。

十一、性能说明

五健专项只有 5 项，切换只是更新 activeHealthCode 和文本内容，不涉及大量数据渲染，不会造成性能压力。
实现时请避免重复生成复杂 DOM，直接基于 healthItems 和 activeHealth 计算展示即可。
