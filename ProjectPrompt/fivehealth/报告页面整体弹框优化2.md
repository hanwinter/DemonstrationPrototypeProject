继续优化五健家长端报告页面中的“五健专项详情底部弹框”。

本次只修改点击“查看详情 >”后弹出的底部详情弹框，不修改报告页主页面结构，不修改五健健康画像卡，不修改人体图 overlay 高亮逻辑，不修改底部导航。

一、当前问题

当前底部弹框已经支持：
1. 五健专项切换；
2. 当前专项详情展示；
3. 弹框固定高度；
4. 弹框内部滚动。

但仍有几个细节问题需要优化：

1. 顶部五健切换页签中同时展示“视力 高、体重 中、脊柱 低、口腔 正常、心理 待填”，信息偏多，页签看起来像状态卡，不够像一级切换导航。
2. 当前专项标题行右侧的大圆形关闭按钮视觉过重，占用空间较大。
3. “异常指标”“健康建议”的内容块与“当前结论”内容在视觉上没有完全左对齐，看起来像“当前结论”的二级内容。
4. “异常指标”“健康建议”的内容块有边线，组件感较重，不够自然。

二、修改目标

1. 顶部五健切换页签只保留专项名称：
   - 视力
   - 体重
   - 脊柱
   - 口腔
   - 心理

2. 页签不再展示风险短状态：
   - 不显示“高”
   - 不显示“中”
   - 不显示“低”
   - 不显示“正常”
   - 不显示“待填”

3. 当前专项标题行继续展示当前专项名称和风险状态：
   - 示例：心理    待填写
   - 示例：视力    高风险
   - 示例：体重    中风险

4. 关闭按钮从“大圆形关闭按钮”改为“轻量小关闭图标”。
5. 同时保留点击遮罩区域关闭弹框的能力。
6. “当前结论、异常指标、健康建议、检查信息”四个模块保持同一级结构，左边界统一。
7. “异常指标”“健康建议”内容块去掉边线，保留浅色背景和圆角。

三、顶部五健切换页签调整

当前类似：

[视力 高] [体重 中] [脊柱 低] [口腔 正常] [心理 待填]

调整为：

[视力] [体重] [脊柱] [口腔] [心理]

要求：
1. 使用 healthItems 循环渲染。
2. 页签内只展示 item.navName。
3. 不再调用 getShortStatus，不再展示状态短字。
4. 当前选中页签高亮。
5. 未选中页签保持浅色背景和灰蓝色文字。
6. 页签横向排列，支持横向滚动。
7. 页签整体作为弹框最顶层一级导航。
8. 点击页签后，继续更新 activeHealthCode。
9. 切换后弹框内容、主页面右侧按钮选中态、人体 overlay 继续同步更新。

推荐结构：

<div class="sheet-health-tabs">
  <button
    v-for="item in healthItems"
    :key="item.code"
    class="sheet-health-tab"
    :class="{ active: item.code === activeHealthCode }"
    @click="activeHealthCode = item.code"
  >
    {{ item.navName }}
  </button>
</div>

四、当前专项标题行调整

页签下方展示当前专项标题行。

推荐结构：

<div class="sheet-header">
  <div class="sheet-title-row">
    <span class="sheet-title">{{ activeHealth.navName }}</span>
    <span class="risk-tag">{{ activeHealth.status }}</span>
  </div>

  <button
    class="sheet-close-light"
    @click="showHealthDetailSheet = false"
    aria-label="关闭"
  >
    ×
  </button>
</div>

要求：
1. 当前标题使用 activeHealth.navName，不使用 activeHealth.name。
2. 风险标签继续展示 activeHealth.status。
3. 风险标签颜色根据状态变化。
4. 关闭按钮不要使用大圆背景。
5. 关闭按钮改成轻量小图标，颜色使用主色或灰蓝色。
6. 关闭按钮点击区域仍然要足够大，建议 32px - 36px。
7. 标题行高度不要过高。

五、关闭交互调整

要求：
1. 点击右上角轻量关闭图标可关闭弹框。
2. 点击弹框外的遮罩区域也可关闭弹框。
3. 关闭弹框时只设置 showHealthDetailSheet = false。
4. 不重置 activeHealthCode。
5. 不改变主页面滚动位置。

注意：
如果当前遮罩点击关闭还没有实现，请补充：
- 遮罩层绑定 @click.self="showHealthDetailSheet = false"
- 弹框主体阻止点击冒泡，避免点击弹框内部误关闭。

六、内容模块左对齐调整

以下四个模块保持并列一级结构：

1. 当前结论
2. 异常指标
3. 健康建议
4. 检查信息

要求：
1. 四个模块都放在同一个 sheet-scroll-body 容器内。
2. 四个模块使用统一的 detail-block 外层。
3. 四个模块标题 block-title 左边界完全一致。
4. 四个模块内容区域左边界完全一致。
5. 不要让“异常指标”和“健康建议”出现额外左缩进。
6. “异常指标”和“健康建议”的背景块宽度与当前结论内容区域一致。
7. “异常指标”“健康建议”不应看起来像“当前结论”的二级内容。

推荐结构：

<div class="sheet-scroll-body">
  <section class="detail-block conclusion-block">
    <div class="block-title">当前结论</div>
    <div class="block-content plain-content">
      {{ activeHealth.description }}
    </div>
  </section>

  <section class="detail-block abnormal-block">
    <div class="block-title">异常指标</div>
    <div class="block-content soft-content">
      {{ activeHealth.abnormalValues.join('、') }}
    </div>
  </section>

  <section class="detail-block advice-block">
    <div class="block-title">健康建议</div>
    <div class="block-content advice-content">
      {{ activeHealth.advice }}
    </div>
  </section>

  <section class="detail-block compact-info-block">
    <div class="block-title">检查信息</div>
    <div class="info-grid">
      ...
    </div>
  </section>
</div>

七、异常指标样式调整

当前“异常指标”内容块有边框，需要去掉边线。

要求：
1. 保留浅白色或浅灰白背景。
2. 去掉 border。
3. 保留圆角。
4. 保持与当前结论内容左边界一致。
5. 内边距建议 12px - 14px。
6. 内容多时自动换行。
7. 字体深色、半加粗即可，不要过重。

推荐 CSS：

.abnormal-block .soft-content {
  padding: 12px 14px;
  border-radius: 12px;
  background: rgba(255, 255, 255, 0.72);
  border: none;
  color: #17313a;
  font-weight: 700;
  line-height: 1.55;
}

八、健康建议样式调整

当前“健康建议”内容块也需要去掉边线，并保持自然浅色提示卡效果。

要求：
1. 保留纯浅青色背景。
2. 去掉 border。
3. 不要使用上下渐变。
4. 保留圆角。
5. 内容块与当前结论左对齐。
6. 宽度与其他模块一致。
7. 内边距建议 14px - 16px。
