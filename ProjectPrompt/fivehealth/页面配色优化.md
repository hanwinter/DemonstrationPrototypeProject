Codex 提示词（可直接使用）

你现在要对现有前端项目中的 “五健个人360档案”页面 进行视觉优化，重点是配色重构，不是重做页面。

一、任务目标

请基于现有页面代码，对“五健个人360档案”页面进行一轮 UI 配色与视觉风格优化，要求：

不改变现有业务功能、数据结构、交互逻辑
尽量不改页面布局结构，只优化视觉表现
页面定位是 儿童青少年五健健康档案
风格要求：
健康、活泼、专业、高级
现代化
可参考 Apple Health / 苹果风格
避免传统医院HIS后台那种“重蓝色、重灰色、硬边框”的感觉
二、优化方向

当前页面存在这些问题，请重点解决：

页面整体配色偏传统医疗后台，颜色生硬
顶部蓝色过重，不够柔和高级
页面灰度太高，显得沉闷、不够儿童健康场景
风险标签（红、橙、黄）过于刺眼，缺少苹果式柔和感
卡片边框感太重，缺乏轻盈感
人体健康画像区域不够精致，和整体风格不统一
三、总体设计风格

请将页面调整为以下风格：

浅背景 + 白色卡片
低饱和度蓝绿系
留白更舒适
圆角更统一
弱阴影替代重边框
状态色柔和但可识别
体现“儿童健康成长档案”的感觉，而不是纯后台管理系统

1）基础色 / 中性色

--bg-card: #FFFFFF;

用于：

顶部导航栏主视觉
选中态 Tab
主按钮
重点链接

不要使用过于刺眼的纯红、纯黄、纯橙。

五、组件级改造要求
1）页面背景
页面整体背景改为 #F5F7FA
不要使用过重的灰色
保持干净、通透
2）顶部导航栏
保留顶部栏结构
顶部主色改为 #5B8FF9
文本使用白色
整体更轻、更现代
不要过于浓重的深蓝医疗风
3）信息卡片
所有主要内容区卡片统一改为：
白底
圆角 16px
弱阴影 0 2px 12px rgba(16,24,40,0.06)
边框尽量弱化或取消
卡片之间保留清晰但柔和的层次感
4）“人体健康画像”区域
保持现有功能和位置
视觉上更精致
如果当前人体是纯蓝色，请改为柔和渐变蓝：
顶部：#A8D8FF
底部：#6BAAF8
支持更轻的科技健康感
该区顶部统计数字保持简洁清晰，数字可用主色和风险色强调
5）Tab / 模式切换按钮（如“医生模式/家长模式”、“二维/3D”等）
选中态：
背景 #5B8FF9
文字白色
未选中态：
背景 #F2F4F7
文字 #344054
圆角建议 8px
风格简洁，接近苹果分段控件 느낌
6）异常指标区域
保持现有内容结构和数据展示逻辑
每条异常指标做成更轻量的“健康事件卡片”样式
风险标签使用柔和胶囊标签：
高风险：#FFF1F1 背景 + #FF6B6B 字
中风险：#FFF8E7 背景 + #FFB547 字
低风险：#EDFAF0 背景 + #34C759 字
标题文字更突出，说明文字弱化
不要大面积红黄冲突
7）历史体检报告区域
保持列表/表格逻辑不变
优化成更简洁、清爽的时间记录风格
日期、异常项、高风险等数字层级更清晰
“查看”按钮或链接使用主色 #5B8FF9
选中 Tab 下划线/高亮用主色
行 hover 状态可增加浅蓝背景 #F5F9FF
8）按钮

统一按钮风格：

主按钮：
背景 #5B8FF9
字体白色
hover #4F83F1
次按钮：
背景 #F2F4F7
文字 #344054
圆角 8px
高度协调统一，不要太硬
9）文字层级

统一字体层级：

页面主标题：#1D2939
二级标题：#344054
普通说明文字：#667085
弱提示文字：#98A2B3
10）分割线
所有线条弱化
分割线统一使用 #EAECF0
尽量用留白替代重边框

抽取统一的颜色变量 / design tokens
