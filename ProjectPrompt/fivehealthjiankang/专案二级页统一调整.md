请基于当前家长端 H5 第二版“五健智行/专案管理”页面，新增并完善以下 5 个二级页面的初版样式：

1. 儿童建档二级页
2. 首诊问卷二级页
3. 知情同意书二级页
4. 复诊计划二级页
5. 家庭训练计划二级页

要求：先生成初版 UI 样式和页面结构，重点保证风格统一、布局清晰、能从当前“流程”页点击进入对应二级页面。暂不要求完整业务逻辑和真实接口。

一、整体 UI 风格要求

1. 二级页面整体风格必须与当前系统二级页一致：
   - 浅青绿色页面背景；
   - 顶部栏保留返回按钮、居中标题；
   - 内容区域使用白色卡片、小圆角、轻量阴影或弱边框；
   - 字体、颜色、间距、圆角与当前专案流程页保持一致；
   - 不要做成后台管理表单风格；
   - 不要使用大面积深色、复杂渐变或过重卡片。

2. 所有二级页统一页面结构：
   - 顶部导航栏；
   - 学生/专案摘要卡片；
   - 当前页面主体内容；
   - 底部固定主操作按钮。

3. 学生/专案摘要卡片建议展示：
   - 林一凡｜男｜10岁
   - 视力健康管理专案
   - 当前阶段：儿童建档 / 首诊问卷 / 知情同意书 / 复诊计划 / 家庭训练计划

4. 一级流程页中的流程节点需要支持点击跳转：
   - 点击“儿童建档”进入儿童建档页；
   - 点击“首诊问卷”进入首诊问卷页；
   - 点击“知情同意书”进入知情同意书页；
   - 点击“复诊计划”进入复诊计划页；
   - 点击“家庭训练记录”进入家庭训练计划页。

二、儿童建档二级页

页面标题：儿童建档

定位：采集儿童基础信息、监护人信息、健康基础信息。

页面结构建议：

1. 学生/专案摘要卡片
2. 表单内容分组：
   - 基本信息
     - 姓名
     - 性别
     - 年龄
     - 身份证号/学籍号
     - 学校
     - 班级
   - 监护人信息
     - 监护人姓名
     - 手机号
     - 与儿童关系
   - 健康基础信息
     - 过敏史
     - 既往病史
     - 家族史
     - 当前用药情况
   - 视力专案信息
     - 是否戴镜
     - 是否做过眼科检查
     - 最近一次视力异常情况

UI要求：
- 使用分组卡片，不要一整页密集表单；
- 输入框、选择项样式要轻量；
- 必填项用小红星或轻提示，不要大面积警示；
- 底部按钮：提交建档信息。

三、首诊问卷二级页

页面标题：首诊问卷

定位：采集儿童用眼习惯、视力表现、既往检查和家庭近视情况。

页面结构建议：

1. 学生/专案摘要卡片
2. 问卷进度区域：
   - 显示“已完成 3/12”或“问卷进度 3/12”
   - 进度展示要轻量，不要做成很重的横向大进度条
3. 问卷题目卡片：
   - 每题一个小卡片或紧凑题目块
   - 支持单选、多选、文本输入样式
4. 示例题目：
   - 孩子每日近距离用眼时长？
   - 每日户外活动时长？
   - 是否经常眯眼看远处？
   - 是否频繁揉眼或眼干？
   - 是否做过验光检查？
   - 父母是否近视？

UI要求：
- 题目之间留白清晰；
- 单选/多选按钮样式要简洁；
- 不要让页面像考试系统，要像家长填写健康问卷；
- 底部按钮：提交首诊问卷。

四、知情同意书二级页

页面标题：知情同意书

定位：阅读、确认、签署专案知情同意书。

页面结构建议：

1. 学生/专案摘要卡片
2. 协议名称卡片：
   - 视力健康管理专案知情同意书
3. 协议正文内容卡片：
   - 服务内容
   - 数据使用说明
   - 家长确认事项
   - 风险与注意事项
4. 重点说明可以使用浅色提示块，但不要过多
5. 底部确认区域：
   - 勾选框：我已阅读并理解以上内容
   - 主按钮：确认签署

已签署状态也需要预留样式：
- 显示“已签署”
- 显示签署时间
- 显示签署人
- 按钮可改为：查看知情同意书

UI要求：
- 不要做成普通文章页；
- 要突出“阅读确认”和“签署动作”；
- 协议正文区域可以滚动，但整体页面不能显得拥挤。

五、复诊计划二级页

页面标题：复诊计划

定位：查看复诊安排，确认复诊计划。

页面结构建议：

1. 学生/专案摘要卡片
2. 复诊通知单样式卡片：
   - 复诊时间：2026-04-18 09:30
   - 复诊地点：眼保健专科门诊
   - 复诊科室：儿童眼保健专科
   - 主治医生：王医生
   - 当前状态：待复诊
3. 复诊项目：
   - 视力复查
   - 屈光检查
   - 眼轴复测
   - 用眼行为评估
4. 注意事项：
   - 请携带既往检查报告
   - 如需散瞳检查，请按医生要求准备
   - 建议家长陪同

UI要求：
- 页面应像“预约/复诊通知单”，不要做成表单；
- 重点突出时间、地点、状态；
- 状态标签“待复诊”可以使用橙色，但不要过大；
- 底部按钮：确认复诊计划；
- 可预留次级按钮：联系机构。

六、家庭训练计划二级页

页面标题：家庭训练计划

定位：展示医生给出的家庭干预任务，并支持家长记录提交。

页面结构建议：

1. 学生/专案摘要卡片
2. 本周训练计划概览：
   - 周期：2026-04-15 至 2026-04-21
   - 计划状态：待提交
3. 训练任务列表：
   - 户外活动：每日不少于 2 小时
   - 远眺训练：每日 3 次，每次 5 分钟
   - 用眼间隔：近距离用眼 30 分钟后休息 10 分钟
   - 眼保健操：每日 1 次
4. 每个任务卡片展示：
   - 任务名称
   - 频次/要求
   - 简短说明
   - 完成状态：未完成/已完成
5. 家长记录区域：
   - 今日是否完成
   - 备注
   - 上传图片入口可先做占位，不需要真实上传逻辑

UI要求：
- 偏任务清单风格；
- 每个训练项用轻量小卡片展示；
- 不要做成纯文本说明；
- 底部按钮：提交训练记录。

七、交互要求

1. 当前流程页点击对应流程节点后，进入对应二级页。
2. 二级页点击顶部返回按钮，返回当前专案流程页。
3. 底部主按钮暂时只做前端模拟：
   - 点击后可显示 toast 或轻提示；
   - 不需要真实接口。
4. 页面状态可以使用 mock 数据。
5. 保持现有首页、报告页、我的页面等其他功能不变。

八、重点检查

请重点检查以下问题：

1. 二级页风格是否与当前专案流程页一致；
2. 顶部栏、卡片、按钮、字体、间距是否统一；
3. 是否避免了大卡片过重、表单过密的问题；
4. 流程页点击跳转是否正常；
5. 返回路径是否正确；
6. 移动端 390px 宽度下是否无横向滚动、无内容挤压、无底部按钮遮挡内容。

先完成初版样式即可，字段和交互可以后续继续优化。
