请继续优化当前家长端 H5 第二版“五健智行/专案管理”中的“知情同意书”二级页面。

本次重点：
1. 增加“未签署 → 勾选确认 → 确认签署 → 已签署”的状态流转；
2. 将页面各卡片纵向间距调整为 5px 进行视觉验证；
3. 优化协议正文区域滚动条样式，避免原生滚动条过重。

一、状态流转设计

请用前端 mock 状态实现，不需要真实接口。

页面需要支持两种主要状态：

1. 未签署状态
2. 已签署状态

二、未签署状态 UI

进入未签署状态时，页面展示：

1. 学生/专案摘要卡片保持原样。
2. 协议标题卡片：
   - 标题：视力健康管理专案知情同意书
   - 右侧状态标签：待签署
   - 状态标签使用浅橙色小胶囊，不要过大。
3. 不显示“签署时间 / 签署人”信息卡片。
   - 或者如果保留信息卡片，也要显示为空状态，但建议初版直接隐藏，页面更干净。
4. 协议正文区域正常展示，可滚动阅读。
5. 正文下方显示勾选项：
   - 我已阅读并理解以上内容
6. 勾选前：
   - 底部主按钮“确认签署”为禁用状态；
   - 按钮颜色弱化，不可点击。
7. 勾选后：
   - 底部主按钮“确认签署”变为可点击；
   - 使用当前系统主青绿色按钮样式。
8. 点击“确认签署”后：
   - toast：知情同意书已签署
   - 状态切换为已签署
   - 自动显示签署时间和签署人

三、已签署状态 UI

已签署状态下，页面展示：

1. 协议标题卡片：
   - 标题：视力健康管理专案知情同意书
   - 右侧状态标签：已签署
   - 状态标签使用浅青绿色或浅绿色小胶囊。
2. 显示签署信息卡片：
   - 签署时间：2026-04-02 10:18
   - 签署人：王女士
3. 协议正文只读展示。
4. 勾选项可以隐藏；
   - 如果保留，则显示为已勾选且不可操作，但建议初版隐藏，避免重复确认。
5. 不再显示“知情同意书已确认”按钮。
6. 底部主按钮改为：
   - 返回专案流程
7. 点击“返回专案流程”：
   - 返回当前专案流程页。

四、底部按钮要求

1. 未签署未勾选：
   - 按钮文案：确认签署
   - 禁用态
2. 未签署已勾选：
   - 按钮文案：确认签署
   - 可点击
3. 已签署：
   - 按钮文案：返回专案流程
   - 可点击
4. 底部按钮保持当前固定底部样式；
5. 底部按钮不能遮挡正文内容；
6. 页面底部要预留安全距离。

五、卡片纵向间距调整

当前页面各卡片之间间距偏大，请将知情同意书页面内主要卡片之间的纵向间距调整为 5px 进行视觉验证。

涉及卡片包括：
- 学生/专案摘要卡片；
- 协议标题卡片；
- 签署信息卡片；
- 协议正文卡片；
- 其他同页面白色内容卡片。

要求：
1. 卡片之间纵向 margin 调整为 5px；
2. 不要同步过度压缩卡片内部 padding；
3. 卡片内部内容仍需保持可读；
4. 删除或避免出现异常空白；
5. 如果有底部固定按钮区域，正文底部仍需预留空间。

六、协议正文滚动条优化

当前协议正文区域右侧原生滚动条较明显，视觉偏重。请进行轻量优化。

要求：
1. 协议正文区域仍然可内部滚动；
2. 滚动条变细；
3. 滚动条颜色弱化，使用浅灰或浅青灰；
4. 滚动条轨道尽量透明或非常浅；
5. 滚动条圆角处理；
6. 不要出现深色粗滚动条；
7. 移动端如果系统隐藏滚动条，也可以保持隐藏；
8. 不要影响正文内容阅读和滚动操作。

可参考样式方向：
- scrollbar-width: thin;
- scrollbar-color: rgba(120, 160, 160, 0.35) transparent;
- ::-webkit-scrollbar width 4px;
- ::-webkit-scrollbar-thumb 使用浅色、圆角；
- ::-webkit-scrollbar-track 透明。
