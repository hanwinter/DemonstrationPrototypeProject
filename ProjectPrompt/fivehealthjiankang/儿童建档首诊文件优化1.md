请继续优化当前家长端 H5 第二版“五健智行/专案管理”中的“儿童建档”二级页面。

本次重点：为儿童建档页增加“查看态 / 编辑态 / 保存 / 提交”的状态设计，区分“保存资料”和“提交建档信息”。

一、优化目标

当前儿童建档页中所有字段看起来都像可编辑输入框，底部只有“提交建档信息”按钮，容易让用户不清楚：
- 当前是在查看资料还是编辑资料；
- 保存和提交是否是一回事；
- 提交后是否还能修改。

请优化为更符合资料型页面的交互：

1. 默认进入页面为“查看态”；
2. 点击“编辑资料”后进入“编辑态”；
3. 编辑态可以“取消”或“保存”；
4. 保存后回到查看态；
5. 查看态下可以点击“提交建档信息”完成建档流程；
6. 提交后页面进入“已提交态”，字段锁定。

二、页面状态设计

请实现以下 4 种状态，可用前端 mock 状态控制，不需要真实接口。

1. 未建档 / 首次填写态
- 如果 mock 数据中建档信息未填写完整，可默认进入编辑态；
- 字段可编辑；
- 底部按钮：提交建档信息；
- 可保留“保存草稿”作为次按钮，如果实现成本低可以加，否则先不加。

2. 已保存未提交：查看态
- 默认展示已填写信息；
- 字段不可编辑；
- 字段视觉应像“信息展示”，不要像强输入框；
- 底部按钮：
  - 次按钮：编辑资料
  - 主按钮：提交建档信息

3. 编辑态
- 字段变为可编辑输入框；
- 必填字段显示小红星；
- 底部按钮：
  - 次按钮：取消
  - 主按钮：保存
- 点击保存后 toast：建档信息已保存，并回到查看态；
- 点击取消后回到查看态，不保存本次修改。

4. 已提交态
- 字段不可编辑；
- 摘要卡片或页面顶部显示“已提交”状态；
- 底部按钮：
  - 主按钮：返回专案流程
  - 可选次按钮：申请修改
- 点击返回专案流程，回到当前专案流程页；
- 申请修改可先 toast：请联系机构申请修改。

三、UI 表现要求

1. 查看态字段样式
- 使用浅底信息框或纯文本行展示；
- 不要显得像正在输入的表单；
- 可以保留轻边框，但要弱化；
- 不显示光标感、输入态高亮。

2. 编辑态字段样式
- 使用当前输入框样式即可；
- 输入框边框和背景可以稍微更明显；
- 下拉/单选/多选字段使用当前系统风格；
- 必填项标题后显示小红星。

3. 页面顶部/摘要卡片状态
在学生/专案摘要卡片中增加状态展示：
- 当前阶段：儿童建档
- 建档状态：已保存未提交 / 编辑中 / 已提交

状态标签样式：
- 已保存未提交：青绿色弱标签
- 编辑中：橙色弱标签
- 已提交：绿色弱标签
标签不要过大，不要抢主标题。

4. 编辑入口
- 不要把“编辑资料”做成很大的主按钮放在页面上方；
- 建议在查看态底部作为次按钮；
- 也可以在摘要卡片右侧放一个小的“编辑”文字按钮；
- 初版以底部次按钮为主即可。

5. 底部固定按钮
- 保持当前底部固定按钮风格；
- 查看态：左侧“编辑资料”，右侧“提交建档信息”；
- 编辑态：左侧“取消”，右侧“保存”；
- 已提交态：主按钮“返回专案流程”；
- 底部按钮不能遮挡页面内容，页面内容底部要预留安全距离；
- 390px 移动端宽度下按钮不能挤压或换行。

四、字段与分组保持不变

继续保留当前儿童建档页已有字段和分组，不要大改字段结构：

1. 基础身份信息
- 儿童姓名
- 性别
- 出生日期
- 年龄
- 身份证号
- 儿童健康档案编号

2. 就读信息
- 就读机构
- 年级
- 班级
- 学籍号

3. 监护人及联络信息
- 监护人姓名
- 与儿童关系
- 联系电话
- 紧急联系人
- 紧急联系人电话
- 户籍地址
- 现居住地址

4. 出生与成长基础信息
- 民族
- 出生孕周
- 是否早产
- 分娩方式

5. 既往健康与家族风险史
- 既往病史
- 手术史
- 长期用药史
- 家族高危史
- 既往体检异常记录
- 以往就诊 / 矫正记录

6. 页面底部入口
- 生活行为基线问卷：未填写 / 已填写
- 五健专项档案概览：待筛查

五、交互要求

1. 点击“编辑资料”
- 页面进入编辑态；
- 字段变为可编辑；
- 状态标签变为“编辑中”；
- 底部按钮变为“取消”“保存”。

2. 点击“保存”
- 显示 toast：建档信息已保存；
- 页面回到查看态；
- 状态标签变为“已保存未提交”；
- 底部按钮变为“编辑资料”“提交建档信息”。

3. 点击“取消”
- 页面回到查看态；
- 状态标签恢复“已保存未提交”；
- 不需要真实恢复数据逻辑，前端 mock 即可。

4. 点击“提交建档信息”
- 显示确认提示或 toast；
- 提交成功后 toast：建档信息已提交；
- 页面进入已提交态；
- 状态标签变为“已提交”；
- 字段锁定；
- 底部按钮变为“返回专案流程”。

5. 点击“返回专案流程”
- 返回当前专案流程页。

六、不要做的内容

1. 不要改动当前专案流程页的整体结构；
2. 不要影响其他二级页；
3. 不要接真实接口；
4. 不要新增复杂权限逻辑；
5. 不要把“编辑资料”做成页面上很重的主视觉按钮；
6. 不要把查看态字段仍然做得像可编辑输入框。

七、验收重点

请重点检查：

1. 查看态、编辑态、已提交态是否能明显区分；
2. 保存和提交的按钮含义是否清楚；
3. 查看态字段是否不再像输入框；
4. 编辑态字段是否清晰可编辑；
5. 状态标签是否轻量，不抢页面主内容；
6. 底部按钮在 390px 移动端宽度下是否不换行、不遮挡内容；
7. 返回当前专案流程页是否正常。
