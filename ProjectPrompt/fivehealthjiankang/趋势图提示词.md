1. 顶部：指标名称 + 趋势标识；

3. 辅助说明：较上次变化；
4. 底部：带检查点位和日期标签的小型趋势图。

三、卡片结构建议

每个卡片建议结构：

- 左上：指标名称，例如“裸眼视力”
- 右上：趋势

- 变化说明：较上次 +0.1
- 小型折线图：
  - 3 个检查点位；
  - 底部显示短日期；
  - 最新点高亮。

示例：

较上次 +0.1

五、小型趋势图 UI 要求

1. 保持当前两列小卡片布局，不要改成大图。
2. 不使用复杂大图表库也可以，推荐用轻量 SVG 或现有前端方式绘制。
3. 每个趋势图至少包含：
   - 折线；
   - 3 个检查点圆点；
   - 最新点高亮；
   - 3 个短日期标签：03-28、04-18、05-16。
4. 日期标签：
   - 字号小；
   - 灰色；
   - 不要拥挤；
   - 不能溢出卡片。
5. 点位样式：
   - 普通点：浅青色小圆点或空心点；

   - 点位大小控制在 4-5px；
   - 不要过大。
6. 折线样式：
   - 主青绿色；
   - 线宽约 2px；
   - 圆角；
   - 不要太粗。
7. 背景辅助线：
   - 可增加 1-2 条极浅横向参考线；
   - 颜色非常浅，例如 #EEF5F4；
   - 不要做完整复杂坐标轴。
8. 图表高度：
   - 控制在卡片底部区域；
   - 不要撑高卡片太多；
   - 保持整体页面紧凑。
