请继续优化当前家长端 H5 第二版“五健智行/专案管理”中的“首诊问卷”二级页面。

本次重点：为首诊问卷增加“填写中 / 保存草稿 / 提交 / 提交后只读”的状态逻辑。注意：首诊问卷是任务型问卷，不要完全照搬儿童建档页的“查看态 → 编辑资料 → 保存”模式。

一、设计目标

首诊问卷需要区分：
- 正在填写；
- 已保存草稿但未提交；
- 已提交；
- 提交后只读查看。

但交互要符合问卷填写习惯：
- 未提交前，用户进入页面后可以直接填写或继续填写；
- 不需要先点击“编辑资料”；
- 支持“保存草稿”；
- 点击“提交首诊问卷”后锁定问卷；
- 已提交后题目只读展示，不可修改。

二、状态设计

请用前端 mock 状态实现以下状态，不需要真实接口。

1. 未填写 / 填写中
- 用户首次进入问卷；
- 题目可直接选择或填写；
- 状态标签显示：填写中 或 未填写；
- 底部按钮：
  - 次按钮：保存草稿
  - 主按钮：提交首诊问卷

2. 已保存未提交
- 用户点击“保存草稿”后进入此状态；
- 问卷内容仍可继续修改；
- 状态标签显示：已保存未提交；
- 底部按钮仍为：
  - 次按钮：保存草稿
  - 主按钮：提交首诊问卷
- 不需要额外增加“编辑资料”按钮；
- 用户可以直接继续填写。

3. 已提交
- 用户点击“提交首诊问卷”后进入此状态；
- 所有题目变为只读；
- 选项不可再切换；
- 输入内容不可再修改；
- 状态标签显示：已提交；
- 底部按钮：
  - 主按钮：返回专案流程
  - 可选次按钮：申请修改

三、摘要卡片状态展示

请参考儿童建档页的状态展示方式，将问卷状态放在摘要卡片中，与“当前阶段”同一行右侧。

推荐布局：
- 左侧：当前阶段：首诊问卷
- 右侧：填写中 / 已保存未提交 / 已提交

状态标签样式：
- 填写中：浅橙色弱标签；
- 已保存未提交：浅青绿色弱标签；
- 已提交：浅绿色弱标签；
- 标签高度小一些，不要抢主标题；
- 不要单独占一行。

四、底部按钮逻辑

1. 未填写 / 填写中 / 已保存未提交
底部固定按钮：
- 左侧弱按钮：保存草稿
- 右侧主按钮：提交首诊问卷

点击“保存草稿”：
- toast：首诊问卷已保存
- 状态标签变为：已保存未提交
- 问卷仍然可继续填写

点击“提交首诊问卷”：
- 可直接提交，暂不要求严格校验；
- toast：首诊问卷已提交
- 状态标签变为：已提交
- 问卷变为只读
- 底部按钮变为：返回专案流程

2. 已提交
底部固定按钮：
- 主按钮：返回专案流程
- 可选弱按钮：申请修改

点击“返回专案流程”：
- 返回当前专案流程页

点击“申请修改”：
- toast：请联系机构申请修改
- 不需要真实接口

五、只读状态 UI 要求

已提交状态下：
1. 选项标签不可点击；
2. 已选选项保持浅青色选中态；
3. 未选选项弱化显示；
4. 输入类问题显示为只读文本或弱化输入框；
5. 不要让用户误以为还能继续填写；
6. 分组仍可展开/收起查看。

六、未提交状态 UI 要求

未填写 / 填写中 / 已保存未提交状态下：
1. 题目可以直接填写；
2. 选项可以正常切换；
3. 输入项可以编辑；
4. 不需要“编辑资料”按钮；
5. 分组结构继续使用“一个问卷分组 = 一张大卡片”的样式；
6. 问题之间继续使用极浅色分割线；
7. 保持当前浅青绿色 H5 风格。

七、问卷进度

继续保留“问卷进度 3/18”卡片。

建议：
- 保存草稿后进度不重置；
- 提交后进度可显示为“已完成 18/18”或保持当前 mock 数据；
- 不需要复杂校验逻辑；
- 初版可按 mock 数据展示。

八、不要做的内容

1. 不要照搬儿童建档页的“编辑资料”按钮逻辑；
2. 不要默认进入只读查看态；
3. 不要把每道题重新拆成独立卡片；
4. 不要接真实接口；
5. 不要影响儿童建档页和其他二级页面；
6. 不要新增复杂必填校验；
7. 不要让已提交状态仍然可以修改选项。

九、验收重点

请重点检查：
1. 首诊问卷是否支持保存草稿；
2. 保存草稿后是否仍可继续填写；
3. 提交后是否进入只读状态；
4. 已提交后选项是否不可再切换；
5. 状态标签是否与“当前阶段：首诊问卷”在同一行右侧；
6. 底部按钮是否根据状态正确切换；
7. 分组大卡片结构是否保持不变；
8. 390px 移动端宽度下是否无横向滚动、无内容遮挡；
9. 底部按钮是否不遮挡最后一道题。
