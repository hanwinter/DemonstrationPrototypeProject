请优化“五健智行专案管理家长端 H5”的“专案”页面 UI。

特别注意：
1. 不要修改当前已经定版的“五健家长端 H5”页面。
2. 本次只优化“五健智行专案管理家长端 H5”中的“专案”页面。
3. 不要影响首页、康复、我的等其他页面的业务逻辑。
4. 当前已定版的原家长端 H5 页面不能覆盖、删除、重构、移动或调整。
5. 保留现有多专案切换、专案生命周期、知情同意按专案维度控制、报告锁定等业务逻辑，只做 UI 与展示结构优化。

整体优化目标：
将当前页面从“卡片和标签堆叠感”优化为“清爽、轻量、层级清晰的专案状态管理页”。

一、卡片样式优化
1. 页面中的卡片不要使用明显边线。
2. 取消或弱化卡片 border，优先使用白底、浅阴影、浅色背景区分层级。
3. 卡片圆角收小：
   - 主卡片建议 10px-12px
   - 小卡片或内部模块建议 8px-10px
4. 不要所有内容都做成强卡片，流程列表、阶段状态这类只读信息要更轻。
5. 页面整体仍保持当前健康、清爽、绿色/蓝绿色风格。

二、滚动条样式优化
1. 页面中的横向滚动条和纵向滚动条样式需要统一。
2. 横向专案切换区域不要显示原生滚动条。
3. 移动端横向滑动可以保留，但通过卡片露出一部分暗示可滑动，不要暴露浏览器默认滚动条。
4. 纵向页面不要显示明显的桌面端粗滚动条。
5. 如果项目中已有统一滚动条样式，请复用整个项目的滚动条规范。
6. 尽量避免页面内部出现多个独立滚动容器，保持整页滚动，底部 Tab 固定即可。

三、专案切换卡片信息减法
当前专案切换卡片中存在信息重复，需要优化。

要求：
1. 专案切换卡片只承担“选择当前专案”的作用，不要承载过多详情。
2. 专案卡片保留：
   - 专案名称
   - 专案类型：视力 / 体重 / 口腔 / 脊柱 / 心理
   - 当前状态：管理中 / 待签署 / 随访中 / 已结案
3. 专案切换卡片中去掉：
   - “已签署”标签
   - “68%”进度百分比
4. 未签署专案卡片可以保留醒目的“待签署”状态。
5. 已签署状态放到下方当前专案详情中展示，不要在专案切换卡片重复展示。
6. 进度百分比统一放到当前专案详情卡片中展示。

四、当前专案详情卡片优化
当前专案详情卡片承担完整信息展示。

要求：
1. 保留专案名称。
2. 展示专科、医生、开始时间。
3. 展示当前阶段。
4. 展示知情同意状态。
5. 展示专案进度。
6. 专案进度需要增加标题，不要只显示百分比。

建议展示方式：
- 左侧：
  - 专案名称
  - 儿童眼保健专科｜主治医生：王医生｜开始：2026-03-28
  - 当前阶段：家庭干预与康复训练
  - 知情同意：已签署
- 右侧或下方：
  - 标题：专案进度
  - 数值：68%
  - 进度条

注意：
1. 不要在详情卡片中堆太多胶囊标签。
2. “管理”“知情同意已签署”可以用轻量文本或小标签，避免抢眼。
3. 进度条保留，但上下间距要紧凑。

五、生命周期步骤条优化
当前数字步骤条可以保留，但需要变轻、变紧凑。

要求：
1. 生命周期用于展示阶段概览，不是操作按钮。
2. 阶段包括：
   - 建档
   - 首诊
   - 授权
   - 管理
   - 复诊
   - 结案
3. 弱化数字圆点，可以改为：
   - 已完成阶段使用小对勾或浅色完成态
   - 当前阶段使用主色高亮
   - 未开始阶段使用灰色
4. 不要让步骤条看起来像可点击按钮。
5. 如果保留数字，也要降低视觉重量。
6. 阶段概览区域不要再做过重边框卡片。

六、Tab 样式优化
当前“流程 / 报告 / 家庭服务”三个 Tab 做成了胶囊按钮，样式偏重，需要调整。

要求：
1. 将 Tab 改成更轻的页签样式。
2. 推荐使用“文本页签 + 下划线”：
   - 流程    报告    家庭服务
   - 当前选中项使用主色文字 + 下划线
   - 未选中项使用普通文本色
3. 不要给每个 Tab 单独加明显边框。
4. Tab 高度控制紧凑，不要占用过多空间。
5. Tab 仍然需要支持切换。
6. 未签署专案的“报告”和“家庭服务”仍需要保持锁定态。

七、流程列表优化
当前流程列表像可点击功能条，需要改成只读流程状态展示。

要求：
1. 流程列表不是功能入口，不要做成一条条可点击卡片。
2. 改成“只读时间轴”或“阶段状态清单”。
3. 推荐使用左侧小圆点 + 竖线的时间轴样式。
4. 每个流程节点包括：
   - 阶段名称
   - 简短说明
   - 状态文本
5. 状态文本要轻量，不要使用过大的胶囊标签。
6. 已完成、进行中、待处理、未开始状态要有区别，但不要过度装饰。
7. 流程列表可以放在一个白底区域内，用分隔线区分每个节点，不要每个节点都做成独立强卡片。

视力专案流程示例：
1. 建档
   - 已完成基础档案与联系方式登记
   - 状态：已完成
2. 首诊
   - 首次专科就诊信息已提交
   - 状态：已提交
3. 授权
   - 该视力专案已完成知情同意签署
   - 状态：已签署
4. 管理
   - 家庭训练与随访进行中
   - 状态：进行中
5. 复诊
   - 下次复诊 2026-04-18 09:30
   - 状态：待复诊
6. 结案
   - 完成阶段评估后结案
   - 状态：未开始

八、未签署专案状态优化
切换到未签署知情同意书的专案时，UI 要清楚但不要过重。

要求：
1. 专案切换卡片显示“待签署”。
2. 当前专案详情中展示：
   - 当前阶段：待签署知情同意
   - 知情同意：未签署
3. 显示轻量提示：
   “该专案尚未签署知情同意书，签署后可查看该专案报告、健康指导和家庭服务。”
4. 提供“去签署知情同意书”按钮。
5. 报告 Tab 和家庭服务 Tab 保持锁定态。
6. 不要使用强烈红色错误提示。
7. 不要写成关闭整个 H5，只限制当前专案个案管理相关内容。

九、页面整体层级建议
优化后的页面结构建议为：

1. 顶部标题
   - 五健智行专案

2. 当前儿童区域
   - 林一凡｜男｜8岁
   - 已参与 2 个专案

3. 专案切换区
   - 视力健康管理专案｜管理中
   - 体重营养管理专案｜待签署

4. 当前专案详情
   - 专案名称
   - 专科 / 医生 / 开始时间
   - 当前阶段
   - 知情同意状态
   - 专案进度 68%
   - 进度条

5. 阶段概览
   - 建档 / 首诊 / 授权 / 管理 / 复诊 / 结案

6. 页签
   - 流程 / 报告 / 家庭服务

7. 页签内容
   - 流程：只读时间轴
   - 报告：专科诊疗报告、LIS 化验单、PACS 检查报告、趋势数据
   - 家庭服务：健康指导、训练记录、随访问卷、在线互动、家长手册
