二、图文报告入口
每条 LIS/PACS 单据都提供“图文报告”入口。

要求：
1. “图文报告”不要做成强主按钮。
2. 推荐使用轻量文字入口：
   - 图文报告 >
   - 或 查看图文报告
3. 点击“图文报告”后，不在当前卡片内展示图片。
4. 点击后进入独立报告查看页面，或打开全屏报告查看层。
5. 不要在列表中展示压缩后的报告图片。

三、独立全屏报告查看器
新增或优化一个“图文报告查看器”。

查看器形式：
优先采用独立页面或全屏预览层，不建议普通小弹框。

查看器顶部：
1. 左侧返回按钮
2. 标题：图文报告
3. 副标题显示：
   - 报告名称
   - 检查日期
4. 示例：
   - 图文报告
   - 维生素D检测｜2026-04-08

查看器内容：
1. 显示对应报告图片。
2. 图片初始状态按屏幕宽度适配。
3. 图片保持原始比例，不要裁剪。
4. 背景可以使用浅灰或白色，突出报告图片。
5. 报告图片过长时允许纵向滚动。
6. 报告图片较宽时允许横向移动。

查看器交互：
1. 支持放大。
2. 支持缩小。
3. 支持拖动移动查看。
4. 移动端支持双指缩放更好。
5. 如果双指缩放实现成本较高，可以先提供简单工具按钮：
   - 放大
   - 缩小
   - 适应屏幕
6. 缩放后允许拖动画面查看局部内容。
7. 支持返回到专案报告页。
8. 查看报告时可以隐藏底部 Tab，避免占用阅读空间。
