下面这段可以直接发给 Codex，用于生成**产程图前端演示页面**。

请基于现有前端项目，生成一个“电子产程图演示页面”，用于孕产保健管理系统的原型演示。页面原则与之前“外院检验报告 OCR 识别校验工作台”一致：只做前端演示，不连接真实后端，全部使用 Mock 数据，重点展示业务流程、页面布局、交互逻辑和系统价值。

一、页面目标

本页面用于演示纸质产程图电子化后的应用效果。传统纸质产程图在实际工作中常存在产后补画、归档为主、实时监测价值不足的问题。电子化后，系统应支持医护人员录入产程观察数据后，在同一页面实时生成和更新电子产程图，并结合预警规则提示产程风险；分娩结束后支持生成标准产程图用于打印归档。

二、页面名称

电子产程图 / 产程监护工作台

三、整体布局要求

采用左右结构或上下结构均可，但需要清晰体现以下区域：

1. 顶部：产妇基础信息区
   显示：

* 姓名：张某某
* 年龄：28岁
* 住院号：ZY20260615001
* 床号：产房03床
* 孕周：39+2周
* 胎次/产次：G2P1
* 入待产时间：2026-06-15 13:00
* 破膜时间：2026-06-15 15:30
* 当前产程状态：第一产程活跃期 / 第二产程 / 已分娩
* 当前风险等级：正常 / 关注 / 预警

2. 左侧或上方：产程数据录入区
   用于模拟医护人员实时录入产程观察记录。字段包括：

* 记录时间
* 宫口扩张 cm
* 胎先露高低 / 胎头下降 station（-5 到 +5）
* 胎心率 bpm
* 宫缩间隔 min
* 宫缩持续时间 sec
* 羊水情况：未破 / 清 / Ⅰ度 / Ⅱ度 / Ⅲ度 / 血性
* 血压
* 脉搏
* 体温
* 备注

按钮：

* 新增记录
* 保存草稿
* 模拟下一小时记录
* 分娩结束
* 打印归档产程图

新增记录后，页面中的电子产程图和观察记录列表需要实时更新。

3. 中央核心区域：电子产程图
   需要用图表形式展示，不要求完全复刻医院纸质表格，但要体现产程图核心逻辑。

图表要求：

* 横轴：时间轴

  * 上方显示相对时间：0、1、2、3……24，表示从入待产或开始观察起经过的小时数
  * 下方显示实际时间：13:00、14:00、15:00、16:00……
  * 支持跨天展示，例如 23:00、00:00、01:00，鼠标悬停时显示完整日期时间
* 纵轴左侧：宫口扩张 0-10cm
* 纵轴右侧：胎先露高低 -5 到 +5
* 宫口扩张用红色圆点和折线表示
* 胎先露高低用蓝色叉号或三角点和折线表示
* 胎心率可在图表下方或独立小图中展示，正常范围 110-160 bpm
* 宫缩频率和持续时间可用简化条形区块展示

图表要体现“录入产程数据后实时刷新”的效果。

4. 右侧：产程预警与辅助判断区
   展示系统根据 Mock 规则自动判断的提示。

预警示例：

* 宫口扩张速度偏慢，请关注产程进展
* 胎心率 172 bpm，超过正常范围，建议复查胎心监护
* 第二产程持续时间接近预警阈值，请关注
* 羊水Ⅱ度污染，请结合胎心情况综合判断

预警级别：

* 正常：绿色
* 关注：黄色
* 预警：红色

5. 下方：产程观察记录列表
   以表格展示每一次录入的数据。

字段包括：

* 记录时间
* 相对小时
* 宫口扩张
* 胎先露高低
* 胎心率
* 宫缩间隔
* 宫缩持续时间
* 羊水情况
* 血压
* 脉搏
* 体温
* 备注
* 操作：编辑 / 删除

编辑或删除记录后，电子产程图需要同步刷新。

五、关键交互

1. 点击“新增记录”

* 将表单数据加入观察记录列表
* 实时刷新电子产程图
* 自动重新计算风险提示

2. 点击“模拟下一小时记录”

* 自动追加一条 Mock 记录
* 图表动态变化

3. 点击“切换正常/异常产程”

* 切换两套 Mock 数据
* 图表和预警同步变化

4. 点击“分娩结束”

* 页面状态变为“已分娩”
* 录入区可置灰或提示已结束
* 显示“生成归档产程图”按钮

5. 点击“打印归档产程图”

* 弹出归档预览弹框
* 弹框中展示标准产程图样式
* 显示产妇基本信息、产程曲线、观察记录
* 提供“打印”“关闭”按钮
* 不需要真正调用打印接口，可以调用 window.print 或仅做演示按钮

六、视觉风格

* 医疗系统风格，简洁、专业、清晰
* 主色调使用蓝色或青蓝色
* 风险预警颜色明确：绿色、黄色、红色
* 页面应适合 1920×1080 分辨率演示
* 信息密度适中，不要太空，也不要拥挤
* 产程图区域要作为页面视觉中心

七、技术要求

* 使用当前项目已有技术栈
* 如果项目是 React，则用 React 实现
* 如果项目是 Vue，则用 Vue 实现
* 图表可使用 ECharts 或当前项目已有图表库
* 不连接后端
* 所有数据写在前端 Mock 文件或组件内
* 保持代码结构清晰，方便后续接入真实接口
* 页面路由可命名为：

  * /delivery/partograph
  * 或 /partograph-demo

八、设计重点

页面要重点体现以下业务价值：

1. 产程图不是单纯的产后归档表单，而是分娩过程中的动态监测工具。
2. 医护人员录入产程观察数据后，系统自动实时更新电子产程图。
3. 系统可根据产程数据进行初步预警，例如宫口扩张停滞、胎心异常、第二产程延长等。
4. 分娩结束后，系统自动生成标准产程图，支持打印归档。
5. 电子化产程图的价值是实现“实时记录、动态展示、风险预警、产后归档”的一体化管理。
