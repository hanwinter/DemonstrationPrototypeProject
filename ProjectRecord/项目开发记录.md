# 项目开发记录

## 1. 项目概况

- 项目用途：管理和展示医院业务前端演示页面
- 当前模块：门诊、高拍仪 OCR、外院报告影像化、产程图、知情同意书、档案补录工作台
- 核心业务：外院报告识别与归档、产程图演示、知情同意书签字归档闭环、档案室批量补录
- 数据来源：全部使用前端 Mock 数据
- 后端连接：无
- 项目目录：`C:\Users\32872\Desktop\jiahufolder\Project_ycf`

## 2. 技术栈

- Vue 3
- JavaScript
- Composition API
- Element Plus
- Vue Router
- Vite
- Vue 单文件组件（SFC）

项目最初的需求文档指定 TypeScript，后续确认改为 JavaScript，以便快速制作和维护演示页面。

## 3. 运行方式

安装依赖：

```powershell
npm install
```

启动开发环境：

```powershell
npm run dev
```

默认访问地址：

```text
http://127.0.0.1:5173/
```

生产构建：

```powershell
npm run build
```

注意：不要直接双击 `index.html`。本项目是 Vite 工程，需要通过开发服务器访问。

## 6. 已实现交互

### 重新扫描

- 显示高拍仪扫描状态
- 模拟等待约 1.8 秒
- 刷新报告图片
- 缩放比例恢复为 72%
- 图片滚动位置恢复到左上角

### 重新识别

- 显示加载状态约 2 秒
- 随机重新生成匹配数量
- 更新识别时间
- 显示成功提示

### 报告图片

- 放大和缩小
- 缩放范围：45% 至 160%
- 支持拖拽移动
- 支持滚动条查看

### 系统项目匹配

- 可搜索下拉选择
- 选择后状态改为“已匹配”
- 来源改为“人工选择”
- 清空后状态改为“未匹配”

### 自动匹配

- 从未匹配记录中匹配约 70%
- 匹配来源改为“规则匹配”
- 显示成功匹配数量

### 新增行

- 新增“人工新增”记录
- 允许编辑系统项目、结果、单位和参考值
- 只有人工新增记录允许删除

### 暂存和导入

- 暂存显示成功提示
- 导入前检查未匹配项目
- 有未匹配项目时弹出确认框
- 确认后显示归档成功提示

## 7. 视觉规范

- 页面风格：医院业务系统，不采用互联网卡片堆叠风格
- 主色：`#447AFC`
- 页面背景：浅灰蓝
- 正文字号：14px
- 重要信息字号：15px
- 区域标题字号：18px
- 左右布局：左侧约 35%，右侧约 65%
- 页面最小宽度：1180px
- 表单标签和控件同行显示
- 表单控件使用固定业务宽度，空间不足时自动换行

## 8. Mock 数据规则

- 默认生成 25 条真实血常规项目
- 初始状态为 20 条已匹配、5 条未匹配
- 已匹配来源包含“规则匹配”和“人工选择”
- 不使用“项目1”“测试项目”等占位名称
- 当前数据包含白细胞、红细胞、血红蛋白、血小板、中性粒细胞等真实检验项目
